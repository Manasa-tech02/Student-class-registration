5. TOKEN MANAGEMENT

   Secure Storage (expo-secure-store)
   - saveTokens(accessToken, refreshToken) → stores both in encrypted storage
   - getAccessToken() → retrieves access token
   - getRefreshToken() → retrieves refresh token
   - clearTokens() → deletes both on logout

   Auto Session Restore (App.tsx on startup)
   - Checks SecureStore for saved tokens
   - Tries GET /auth/me with access token
   - If 401 → tries POST /auth/refresh with refresh token
   - If refresh works → saves new tokens → goes to dashboard
   - If both fail → clears tokens → shows login screen
   - Outcome: User reopens app and lands on dashboard without re-login
     (as long as refresh token is valid — up to 7 days)

   Auto Token Refresh (api.ts withAutoRefresh)
   - Wraps any API call
   - If call returns 401 → automatically tries refresh
   - If refresh succeeds → retries original call with new token
   - If refresh fails → triggers session expired → back to login
   - Outcome: User never sees "session expired" during normal use

   Session Expired Callback
   - setOnSessionExpired() registers a global callback
   - Called when refresh token is also expired
   - Clears all state and navigates to login
   - Outcome: Clean logout when tokens are truly expired

6. API LAYER (api.ts)

   Auth Functions
   - signup(input) → POST /auth/signup → returns { token, refreshToken }
   - login(input) → POST /auth/login → returns { token, refreshToken }
   - refreshTokens(refreshToken) → POST /auth/refresh → returns new pair
   - getMe(token) → GET /auth/me → returns { user }

   Course Functions
   - getCourses(token) → GET /courses → returns { courses }

   Enrollment Functions
   - enrollInCourse(token, courseId) → POST /enroll → returns { message }
   - dropCourse(token, courseId) → DELETE /enroll → returns { message }
   - getMyCourses(token) → GET /enroll/my-courses → returns { courses }

   Error Handling
   - Parses API error responses into user-friendly messages
   - Extracts Zod field validation errors (e.g. "email: Invalid email")
   - Maps HTTP status codes to readable messages
   - Outcome: User always sees a clear, helpful error message

7. NAVIGATION

   State-based routing in App.tsx (no external library)
   - 5 screen states: login | signup | dashboard | courses | courseDetail
   - Conditional rendering based on current screen state
   - Props-based navigation callbacks between screens
   - Auth guard: dashboard/courses/courseDetail require token + user
   - Outcome: Lightweight navigation without react-navigation dependency
